Git Collaboration Works

원격 저장소를 쓰는 이유 : 다른 사람과 협업을 위함

로컬 저장소에 지정한 원격저장소(http://github.com/bellchoi/test-git.git)를 연결

git remote add origin [원격저장소주소] 에러 발생시 : git remote set-url origin https://bellchoi@github.com//bellchoi/test-git.git

git이 관리하는 로컬저장소의 파일들을 원격저장소에 동기화

git push [원격저장소 별칭(origin)] [브랜치이름;main]

깃허브에 있는 원격저장소(최신파일 포함)에 있는 것을 로컬 저장소로 받아 오는 방법

본 로컬 저장소와 연결된 원격저장소에서 커밋과 파일을 받아온다 git fetch 입력 -> fetch로 받아온 정보는 기존 branch에 저장하는 것이 아니고 renmote branch라는 특수한 곳에 저장 됨 ; 확인 명령 git branch -a를 치면 'remote/origin/main'곳에 저장됨 ==> 즉, fetch를 하면 여기에 저장되기 때문에 로컬브랜치에는 저장이 안된 상태임. -> 작업중이던 로컬 브랜치에 연결해 줄려면 ; git merge origin/main => git fetch + merge ===> git pull [원격저장소 이름 :origin][작업중인 브렌치 :main]

다른 사람의 원작자 원격저장소의 파일을 내 원격저장소로 복사해 오기 : fork

14:00에 복제 해온 후 다른 사람이 작업한 14:30의 commit 18시에 추가된 파일은 내 원격 저장소에는 반영되 않고 내가 만든 commit, 내가 추가한 파일은 원작자의 원격저장소에는 반영됮 않음

원작자의 파일을 가지고 내가 수정한 후 수정된 파일을 원작자에게 보내려고 한다면 : pull request

pull request: 내가 수정한 사항ㅇ을 원작자의 원격저장소에 반영되도록 요청하는 것 원작자가 승인 /거절을 할 수 있다. -> 승인한 경우 본 프로젝트에 "contribution"헀다고 나옴

issue : 본 소프트웨어에서 발견한 버그를 보고하거나 질문이 있는 경우 개발자들끼리 소통할 수 있는 기능

git branch [브랜치 이름] -> 지정한 이름으로 branch 생성

브랜치 이동 : git checkout 브랜치명
